# Workflow Snapshot

## Core workflow areas

Nominas is best understood through the workflows it supports.

## 1. Weekly salary capture

The system supports the weekly capture of labor by:

- employee
- project
- work date
- compensation setup

This replaces spreadsheet-heavy weekly reporting flows and gives the team a consistent operational path.

## 2. Payroll generation

Captured weekly labor feeds payroll-related flows:

- pay-period detail
- tax calculation
- payroll generation
- downstream reporting

The value here is not just payroll math. It is keeping labor capture and payroll outputs aligned.

## 3. Financial control

A separate layer tracks weekly project financial control:

- labor subtotal
- admin fee
- adjustments
- requested amount
- paid amount
- remaining balance

This gives project-cost visibility beyond raw payroll entries.

## 4. Reporting

The product includes reporting because operational teams need to:

- review weekly activity
- export data
- track project-level cost status
- reduce repeated manual reconciliation

## 5. Role-based internal access

The system supports multiple internal roles so that:

- admins control setup and sensitive actions
- supervisors operate workflows
- viewers can review information safely

## Portfolio interpretation

From a portfolio perspective, Nominas demonstrates:

- internal product design
- operational workflow modeling
- full-stack delivery
- business-facing systems thinking
